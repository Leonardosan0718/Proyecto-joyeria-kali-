# Joyería Digital (Joyeria Kaligrafi) - Sistema de Navegación por Estilos Estéticos

## 📌 Visión General del Proyecto
[cite_start]Este proyecto forma parte del **Proyecto Integrador Dual**. Consiste en una plataforma de comercio electrónico especializada en joyería, diseñada para resolver la dificultad de localización de productos específicos mediante un sistema de **filtrado por estilos estéticos** (Minimalista, Vintage, Bohemio).

El desarrollo se divide en dos vertientes obligatorias:
1.  [cite_start]**The Job (Ingeniería):** Construcción de un MVP (Producto Mínimo Viable) funcional con arquitectura profesional.
2.  [cite_start]**The Science (Investigación):** Validación empírica de una hipótesis mediante la recolección y análisis de datos de usuario.

## 🧪 Rigor Científico: Hipótesis
Para cumplir con la vertiente de investigación aplicada, el software está instrumentado para validar la siguiente hipótesis causal:

> *"La implementación de un sistema de filtrado por estilos estéticos reducirá el **tiempo promedio de localización de producto (Time-on-Task)** en un **30%** en comparación con la navegación tradicional por categorías generales."*

### Instrumentación (Carpeta `/logs`)
El sistema genera automáticamente archivos en formato `.csv` que registran:
* ID de sesión anónima.
* Timestamp de inicio de búsqueda.
* Filtros aplicados.
* Timestamp de selección de producto final.

## 🛠️ Stack Tecnológico
Para garantizar la solvencia técnica, se utiliza el siguiente stack:
* **Frontend:** React (Interfaces dinámicas y componentes reutilizables).
* **Backend:** Python con FastAPI (Procesamiento de lógica y API REST).
* **Base de Datos:** SQL Server (Persistencia de datos relacional).
* **IDE:** Visual Studio Code.
* [cite_start]**Control de Versiones:** Git (GitHub/GitLab) para trazabilidad de trabajo.

## 📂 Estructura del Repositorio
* `/frontend`: Código fuente de la interfaz de usuario en React.
* `/backend`: Lógica de negocio y Endpoints en Python.
* `/database`: Scripts SQL para la creación y gestión de tablas en SQL Server.
* `/logs`: Datos recolectados para el análisis estadístico de la hipótesis.
* [cite_start]`/docs`: Documentación técnica y académica bajo normas APA.
